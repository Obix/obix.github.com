About the author


Christian Neumanns


Welcome!

My name is Christian Neumanns and I dream of 'number of bugs = 0'.

After graduating as an IT engineer I've been developing customized ERP software for over 20 years, using dBase 3 in the early days, and then, since 1995, Powerbuilder. Moreover, I've written some desktop and web applications in Java and other programming languages.

When I started to develop ERP software, I ignored a very important rule related to software development - the Fail fast! principle. Instead, I applied a Forgive and hope that everything will be fine! principle. The (unanticipated) consequences were inevitable: not-very-happy customers, frustration and loss of time and revenue. After long sessions of pondering I finally decided to develop a programming language that should help to write more reliable code in less time. The result is Obix. The Fail fast! rule is embedded and consistently applied in Obix, not only in the language, but also in its libraries and the framework. Therefore more bugs are automatically detected at compile-time or immediately at run-time.

I like to read beautiful code and I ardently try to write beautiful code. I believe that high quality code can best be achieved when self-motivated and goal-oriented people work together in a friendly open source community. Contributing to open source makes people feel good.

I am a member of:

  • Association for Computing Machinery (ACM)
  • Special Interest Group on Programming Languages (SIGPLAN).
  • IEEE Computer Society

Currently I live in Thailand because in Thailand people smile a lot and they ask you: "Are you happy today?".

In my free time I like reading non-fiction books, traveling, having fun with nice people, and walking in the nature while pondering about new techniques to catch more bugs.
