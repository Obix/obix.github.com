What we could do ...

There are a number or commonly used approaches for creating application user interfaces. For example we could:

  • write a command line utility

    In our case we could write code to interpret and execute the following two commands a user would type in a terminal window:

    1. to add a contribution:

      add_contribution first_name:"John" last_name:"Crawford" random_number:7

    2. to create the 'random arcs' picture:

      create_picture show_contributor_names:yes

  • create a web application using a single technology or a combination of:

    • HTML, Javascript and CSS
    • Java's servlet or JSP technology
    • a Java web framework

    For more information please refer to the section called “Developing a web application”

  • write a graphical user interface (GUI) for a desktop, touchpad or mobile phone with menus, forms and buttons


Besides developing interfaces for people we could also develop interfaces for other applications (possibly written in different programming languages) which can then execute 'random arcs' commands. For example we could:

  • write a HTTP or socket server so that commands could be executed through a HTTP or socket request sent by third-party applications (e.g. a mobile phone application)

    for instance, the following request could be send to a 'random arcs' HTTP server to add a contribution:

    http://www.your_company.com/random_arcs/add_contribution?first_name=John&last_name=Crawford&random_number=7

  • provide the possibility to import commands stored in text files

    for instance, the following commands could be read and executed from a text file generated by another application:

    add_contribution first_name:"John" last_name:"Crawford" random_number:7
add_contribution first_name:"Lisa" last_name:"Simpson" random_number:1
create_picture

  • use the operating system's piping facility to exchange data through the system's standard input/output/error devices


It goes without saying that creating anyone of the above interfaces requires a considerable amount of development time. The good news is that we don't have to program these interfaces manually, as explained in the next chapter.
