Preface

Welcome!

This tutorial is about starting to program in Obix. It is a step by step introduction based on learning by example.

The tutorial is intended for people who have a basic experience in writing programs for computers. Having a basic knowledge of the Java programming language is an advantage (but not a requirement), because Obix creates applications for the Java runtime environment.

If you want to practice the exercises (highly recommended) you can download Obix free of charge at www.rps-obix.com.

This tutorial is divided into the following parts

  1. Simplicity

    Developing high quality software is not easy. But it should at least be as easy as possible.

    In this part of the tutorial you will see how easy it is to quickly build a very common type of application - a web application with data stored in a database.

  2. Versatility

    Obix supports different ways to write and execute code, as well as different kinds of applications. You can create standalone script files, standard PC applications, command line utilities, web applications and more. You can also embed Java source and binary code in your Obix project. Each case will be illustrated by writing the famous Hello world program.

  3. Reliability

    Writing reliable code is the number one goal in Obix. Obix has been designed from the ground up with this goal in mind.

    The intention of this part is to give you an overview of the basic concepts integrated in Obix and, most importantly, the concepts that help creating more reliable software. You will see how to apply these concepts in day to day programming.

  4. Productivity

    Nobody wants to spend time with boring boiler-plate-code or with recurring tasks that could well be accomplished by libraries and frameworks.

    In this part you will be introduced to a prototype of the Practical Application Input/Output framework. This framework is a standard component included in the Obix distribution. It has been developed to minimize the code (and in some cases to completely eliminate the code) you have to write for creating application interfaces like web pages, command line utilities and application-to-application communication. Thus, you can focus on writing the fun part of your application - the so-called 'business logic'.


To explore the full power of Obix and get more information, you can refer to the following resources:

  • the examples in Obix's examples/ subdirectory which is included in the Obix distribution.

  • API (Application Program Interface) browser

  • language manual

  • EBNF (Extended Backus-Naur Form) documentation

If you still need more information then please visit the community page.

Good luck!

Enjoy programming with Obix!
