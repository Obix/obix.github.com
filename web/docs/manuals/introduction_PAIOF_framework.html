While the previous part of this tutorial focused on how to write reliable code, this part will focus on increased developer productivity. This is the second main goal pursued with Obix.

Application input output interfaces

In the last chapter we created the 'random arcs' application. The objective was to make our code as reliable as possible. Therefore we used some of Obix's 'Fail fast!' techniques and we wrote unit tests to ensure that everything works fine.

What remains to be done is to create one or more interfaces to our application so that it can easily be used by people and by other applications.
