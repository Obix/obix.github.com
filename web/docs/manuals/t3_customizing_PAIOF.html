Customizing PAIOF

An important characteristic of any software framework is its ability for being customized in order to meet specific needs which can vary largely in different projects.

PAIOF has been designed from the ground up with highest customization in mind. It is modularized in small components which all have a single responsibility. Every component can programmatically be replaced for maximum flexibility. For example you could provide your own HTML writers to produce specific HTML code. Or you could provide your own command executor if you want code to be executed before or after command execution (for example for logging purposes). The goal of this tutorial is to just get an overview. Please refer to PAIOF's documentation if you need more information.

One obvious improvement in our 'random arcs' application would be to provide a more user-friendly interface. For example:

  • instead of displaying command name add_contribution (which is simply the service command's identifier in the source code), it would be better to display something like Add a new contribution, and provide a short help text

  • instead of displaying random_number it would be more user-friendly to display Random number for radius (1..15) and provide a tool tip when the user hovers over the input field

  • it would be useful to provide alternative short names for commands and parameters. Thus instead of typing

    run add_contribution first_name:"foo" last_name:"bar" random_number:15

    in a terminal window you can simply type:

    run ac fn:"foo" ln:"bar" rn:15

To see how this can be done please refer to project random_arcs which is included in the examples directory of your Obix installation. You can start the web server for this project with run @sws (as explained in the previous section) and have a look at the user-interface in your browser. You can then study and edit the source code to get familiarized with a first level of PAIOF customization.
