Summary of development steps

We start this tutorial by writing one of the most typical applications created in the world of software development: a web application that allows users to manage data stored in a database; a so-called CRUD application (Create, Read, Update and Delete).

We will keep things simple and just use a flat data structure: a list of tasks to be done - a to-do list.

Each task is composed of:

  • a unique identifier, also called a primary key (an auto-incremented integer number: 1,2,3,...)

  • a title (single-line text)

  • a description (multi-line text)

  • the task's priority (low, medium or high)

  • the data and time of creation of the task

  • a flag indicating if the task is done

The steps involved to develop our application are:

  • creating a new project

  • writing source code for task objects

  • adding persistence to store task objects in a database

  • creating a web interface to manage tasks

  • bonus 1: adding a web interface to directly manipulate database tables

  • bonus 2: creating a command line utility to manage tasks by typing commands in a Linux or Windows terminal window


To keep this introduction short and save your time, explanations in this chapter will be kept to a minimum. The goal of this first exercise is to demonstrate how easy it is to write a working CRUD application in minutes, instead of spending hours or days. The code you'll write (without unit testing code) will consist of fewer than 20 instructions!

Let's start!

Step 1: Creating a new project

To create a new project proceed as follows:

  • If not done already, install Obix (it's just a single compressed file to download and uncompress into a directory of your choice).

  • Open a terminal window in your operating system (Linux, MacOS X, Unix or Windows).

  • Go to any directory in which you want to create your new Obix project

  • Create a new project named to_do_list by typing the following command: 

    obix create_project project_id:to_do_list

    Obix creates a subdirectory named to_do_list in your current directory. This subdirectory contains all files needed to manage your project.
