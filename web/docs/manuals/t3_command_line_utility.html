Creating a command line utility

The first example shows how PAIOF helps to leverage any application into a command line utility.

To add a contribution from the command line, open a command line window in your project's root directory (random_arcs) and type:

on Linux:
./run.sh add_contribution first_name:"Nicolas" last_name:"Paganini" random_number:15
on Windows:
run add_contribution first_name:"Nicolas" last_name:"Paganini" random_number:15

After executing the command the result is displayed in the terminal window:

Result: Nicolas Paganini contributed '15' on 2012-05-17 14:16:41
Note
We will later see how to provide short names for commands and parameters so that instead of typing: 
add_contribution first_name:"Nicolas" last_name:"Paganini" random_number:15
you can simply type: 
ac fn:"Nicolas" ln:"Paganini" rn:15

To create the SVG picture type:

on Linux:
./run.sh create_picture show_contributor_names: yes
on Windows:
run create_picture show_contributor_names: yes

The output will be like this:

Result: Ok. The picture has been stored to file F:\obix\projects\random_arcs\picture.svg

Because input argument show_contributor_names has a default value of no, you can leave off this parameter on the command line (if you don't want to show the contributor's names in the picture):

on Linux:
./run.sh create_picture
on Windows:
run create_picture

The above examples show how PAIOF is useful to quickly create command line utilities. The obvious advantage is that you don't have to write yourself a command line parser. Moreover, PAIOF automatically takes care of error handling. Just try to type invalid instructions and see how PAIOF carefully checks and rejects all invalid data.

The versatility of PAIOF, however, comes from the fact that you can combine all of the following features in any ways you like:

  • read input parameters from 10 different sources

  • specify output options:

    • 10 different output destinations
    • 4 different output formats
    • 5 different output content levels

  • execute a list of commands (instead of just a single command)

  • use an integrated web server to instantly create a graphical web interface or a HTTP server for other applications
