In the previous chapters you saw how to create, compile and deploy different kinds of applications. To keep things simple you just wrote a simple 'Hello world' application. It is now time to write a more 'real' application.

Besides showing some basic principles used in day-to-day programming, the focus in this part of the tutorial is to get you in touch with those concepts that help to achieve the main goal of Obix, namely to write reliable and robust code. You will also see how Obix helps you to get more productive.

After reading (and exercising) this part of the tutorial you should be able to develop your own applications in Obix.

Note

If you are not yet familiar with the fundamental concepts of types, factories, and services in Obix then please first read Part I, “Fundamental concepts” in the programming language manual. This takes just a few minutes.

The 'Random arcs' application

We are now going to write an application that creates a picture based on random data provided by people. The picture will be composed of adjacent 90 degrees arcs with different radii and colors. Here is an example of the final result:

Figure 3.1. Obix programming language random arcs tutorial example


The arcs' radii will be based on data provided by users of the application. A single user input for one arc is called a contribution and is composed of:

  • a numerical identifier (autoincrement)
  • the 'first name' and 'last name' of the contributor
  • a random value used to calculate a radius, ranging from 1 to 15
  • the date and time the contribution was submitted

All contributions will be saved in a JSON text file.

The arcs' colors will be defined by a random generator.

The picture will be stored in a Scalable Vector Graphics (SVG) file.

The application will be able to collect contributions directly submitted by users (e.g. via a web site) and/or by other applications (e.g. via a HTTP request).


To develop this application you will do the following:

  1. Create a new application
  2. Write code to create contribution objects
  3. Write unit tests (to detect bugs)
  4. Make data more reliable and secure (!)
  5. Convert objects to (and re-create them from) JSON
  6. Use files to store/restore data
  7. Create the 'random arcs' picture and save it in a SVG file
  8. Run the application with test data (and hopefully enjoy beautil drawings)

Note
This application clearly doesn't belong to the category 'life saving applications'. We will nevertheless write the most reliable code possible, because that is the primary goal of Obix. User input data can be invalid or malicious, and resources like files can fail at runtime. We have to anticipate all kinds of runtime problems and handle them appropriately. This is a challenge, but mandatory in the world of high quality software development. At the end of this chapter you will have a good overview of Obix's intergrated features to help writing more reliable code in less time.
Note
The full project with source code is included in the Obix download. It is located in subdirectory examples/random_arcs/.
